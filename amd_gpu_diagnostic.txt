  ROOT CAUSE ANALYSIS
============================================================

ISSUE 1 - SERVER STARTUP FAILURE (ModuleNotFoundError: No module named 'codebase_indexer')
  The RUNNING container was built without docker-compose.amd.wsl2.yml overlay.
  The LATEST image correctly resolves codebase_indexer from /app/src (confirmed in [3/6]).
  FIX: Rebuild and restart with:
    docker compose -f docker-compose.yml -f docker-compose.amd.wsl2.yml up -d --build

ISSUE 2 - librocdxg.so MISSING on WSL2 host
  docker-compose.amd.wsl2.yml bind-mounts /opt/rocm/lib/librocdxg.so into the container.
  This file does NOT exist on the WSL2 host (/opt/rocm/lib/librocdxg.so: NO).
  Docker will create it as an empty directory (causing a broken mount) or silently skip it.
  This is why rocminfo reports "Cannot load librocdxg.so".
  FIX: Install ROCm 7.2.1+ on the Windows host (installs librocdxg.so under /opt/rocm/lib
       inside WSL2), or make the volume mount conditional (see fix applied below).

ISSUE 3 - libmigraphx_c.so.3 MISSING in container
  The onnxruntime_migraphx provider requires the MIGraphX runtime library.
  libmigraphx_c.so.3 is not bundled in rocm/dev-ubuntu-24.04:7.2.1 base image.
  MIGraphXExecutionProvider is listed by ORT but CANNOT run GPU inference.
  ORT will silently fall back to ROCMExecutionProvider (if GPU is accessible) or CPU.
  FIX: Install migraphx package in the Dockerfile builder-rocm-wsl stage:
    RUN apt-get install -y migraphx  (if available) or install from ROCm apt repo.

ISSUE 4 - fastembed / onnxruntime NOT in uv venv
  'uv pip list' is unavailable (pip not installed in uv venv by default).
  ORT detected in [4/6] is likely from system Python, not the .venv.
  fastembed must be in .venv for the MCP server to embed chunks.
  FIX: Verify 'uv sync --frozen' installed fastembed. Check with:
    docker run --rm <image> /app/.venv/bin/python -c "import fastembed; print(fastembed.__version__)"
